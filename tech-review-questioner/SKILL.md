# 技术方案深度追问生成器

## 任务目标

对用户提供的技术方案文档，从多个维度生成大量追问问题，模拟"打破砂锅问到底"的评审风格。问题要求：数量多、角度刁钻、逻辑连贯、层层递进。

## 输入

用户提供技术方案文档的完整内容或关键章节。

## 输出格式

生成所有问题后，按类别分组输出，每组标注问题数量。确保：
1. 问题之间有逻辑递进关系
2. 每个问题简洁明了，直击要害
3. 数量不少于 80 个问题
4. 覆盖方案的全生命周期

## 注意事项

- 保持追问风格一致：质疑、追问原因、追问替代方案
- 不要假设答案，让被提问者自己思考
- 问题要具体，不要过于笼统
- 适当追问团队协作和外部依赖
